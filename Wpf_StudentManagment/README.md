# 🧑‍🎓 Менеджер студентов — WPF обучающий проект

Добро пожаловать!  
Этот проект создан в **обучающих целях** для демонстрации основных возможностей **WPF** и паттерна **MVVM**.  
Он идеально подходит для студентов, которые уже знакомы с WinForms и хотят понять, зачем нужны XAML, Binding, команды и стили.

Проект реализует простое приложение для добавления и просмотра списка студентов. Весь интерфейс полностью отделён от логики, что наглядно показывает преимущества современных UI-фреймворков.

---

## 🎯 Цели проекта

- Изучить **XAML** и основные контейнеры разметки
- Понять **Data Binding** (связывание данных)
- Освоить **MVVM** (Model-View-ViewModel)
- Научиться использовать **команды (Commands)** вместо событий
- Применить **стили (Styles)** для единого оформления
- Увидеть автоматическое обновление интерфейса через `ObservableCollection`

## 🧱 Структура проекта

Проект построен по принципу MVVM, что чётко разделяет интерфейс и бизнес-логику:

```

├── Views/
│   └── MainWindow.xaml       # Интерфейс (окно приложения)
│
├── Resources/
│   └── Styles.xaml           # Стили и кисти
│
└── App.xaml                  # Подключение ресурсов
```

## 🧠 MVVM в этом проекте — как это работает?

### Model (Модель) — `Student.cs`
Хранит данные студента. Реализует `INotifyPropertyChanged`, чтобы при изменении имени или возраста интерфейс обновлялся автоматически.

### ViewModel — `MainViewModel.cs`
Содержит всю логику:
- `ObservableCollection<Student>` — список студентов (при добавлении нового — таблица обновляется сама)
- `StudentName`, `StudentAge` — поля для ввода
- `AddStudentCommand` — команда, вызывающая метод `AddStudent`
- Метод `AddStudent` добавляет нового студента в коллекцию

### View — `MainWindow.xaml`
Только интерфейс:
- Элементы управления привязаны к свойствам `MainViewModel`
- Кнопка привязана к команде `AddStudentCommand`
- Таблица привязана к коллекции `Students`

> 💡 **Ключевой момент:** В файле `MainWindow.xaml.cs` нет кода — только строчка `DataContext = new MainViewModel();`. Вся логика вне представления.

## 🔍 Сравнение с WinForms

| WinForms                          | WPF в этом проекте                       |
|-----------------------------------|------------------------------------------|
| `button.Click += ...`             | `Command="{Binding AddStudentCommand}"` |
| Нужно писать `dataGrid.Refresh()` | `ObservableCollection` обновляет UI сам  |
| Код в `code-behind`               | MVVM — код в `MainViewModel`             |
| Ограниченная кастомизация         | Гибкие стили в `Styles.xaml`             |
| Сложная работа со списками        | `DataGrid` + `Binding`                   |

## 🧪 Как использовать этот репозиторий как шпаргалку

### 1. Найти нужный элемент
Открой `MainWindow.xaml` — там есть практически все стандартные элементы WPF с привязками.

### 2. Посмотреть, как работает Binding
Например, `Text="{Binding StudentName}"` — обновляется автоматически при вводе.

### 3. Вспомнить команды
Посмотри `RelayCommand.cs` и как `AddStudentCommand` используется во `View` и `ViewModel`.

### 4. Поменять стили
Открой `Resources/Styles.xaml` — измени цвет кнопки или радиус рамки, и изменения применятся ко всем элементам.

### 5. Добавить новую функцию
Вот как можно расширить проект:
- Добавить кнопку **Удалить** → новая команда в `MainViewModel`
- Добавить **Выбранный студент** → свойство `SelectedStudent`
- Подключить **Сохранение в JSON** → пара строк в `MainViewModel`
